# 美术资产需求清单

## 1. 文档用途

本文件用于整理《创世纪》Demo 当前阶段所需的美术资产。

本文件只回答以下问题：

- 这版 Demo 现在需要哪些美术资产
- 哪些资产必须先做
- 哪些资产可以后置
- 美术组当前应先开哪些包
- 后续资产台账按什么字段维护

本文件不展开制作技法，不替代排期表和资产台账。

---

## 2. 当前资产范围判断

本版 Demo 当前边界为：

- 15～20 分钟体验
- 1 个主舞台
- 1 条主流程
- 1 次关键冲突
- 少量核心界面
- 内部验证优先

因此，美术资产以**主流程可跑、舞台可认、界面可用、反馈明确**为标准，不按正式版体量准备。

---

## 3. 当前必须要有的美术资产

## 3.1 主舞台场景资产

当前必须具备：

- 中央广场主场景整体视觉
- 主地面 / 主通路
- 核心建筑立面或背景轮廓
- 远景 / 中景 / 近景分层
- 标语、公告栏、旗帜、导视牌
- 蝶茧终端在场景中的实体表现
- 可交互点位外观
- 基础封边资产
- 基础场景装饰物

当前要求：

- 能立住主舞台识别度
- 能看出制度化环境
- 能区分可走区域与不可走区域
- 能标出关键交互点

---

## 3.2 角色资产

当前必须具备：

- 关键引导对象 1 名
- 功能型 NPC 若干
- 冲突对象 / 敌对对象 1 组
- 玩家在界面中的身份表现素材（如头像 / 牌面）

按表现方案至少满足其一：

- 对话立绘
- 场景角色形象
- 头像 / 剪影 / 占位形象

当前要求：

- 优先保功能角色
- 优先保关键对象识别度
- 不先扩大量次级角色差分

---

## 3.3 UI 资产

当前必须具备：

- 主菜单 / 开始界面基础视觉
- 探索界面基础 HUD
- 对话框、名字框、选项按钮
- 检定 / 选择反馈样式
- 事件弹窗 / 结果页框体
- Demo 结束界面
- 通用按钮、页签、提示框、滚动条、关闭按钮
- 状态图标、权限图标、锁定图标、警示图标

当前要求：

- 风格统一
- 信息可读
- 状态可区分
- 可直接接入流程

---

## 3.4 蝶茧终端专项资产

当前必须具备：

- 终端首页
- 公共公告页
- 权限状态页
- 限制提示页
- 事件反馈页
- 终端专属图标系统
- 终端风格底纹 / 边框 / 分割线 / 状态标签
- 权限等级视觉区分
- 异常 / 禁止访问 / 校验失败反馈样式

当前要求：

- 与普通 UI 明确区分
- 有统一制度化视觉
- 能体现接入、权限、限制三类状态

---

## 3.5 事件与反馈资产

当前必须具备：

- 调查成功 / 失败提示
- 检定成功 / 失败提示
- 权限不足提示
- 新信息获取提示
- 新目标更新提示
- 事件触发弹窗
- 成功 / 绕路 / 替代推进结果页
- 尾钩收束页表现

当前要求：

- 玩家能看懂发生了什么
- 玩家能看懂当前结果
- 玩家能看懂下一步去哪里

---

## 3.6 冲突 / 战斗资产

### 如保留真实战斗，当前必须具备：
- 战斗场景背景
- 我方表现单位
- 敌方表现单位
- 技能图标
- 状态图标
- 目标选中反馈
- 受击 / 命中 / 状态变化基础特效
- 战斗结算页

### 如不保留真实战斗，当前必须具备：
- 冲突结果页
- 风险事件表现图
- 成败反馈样式
- 关键对象 / 敌对对象事件表现图

---

## 3.7 通用特效与占位资产

当前必须具备：

- 交互高亮
- 可调查点提示
- 权限锁定提示
- 页面切换过渡
- 基础点击反馈
- 缺失资源占位图
- 开发期临时占位框 / 占位底图
- 默认图标或默认卡面

当前要求：

- 缺资源时不断流程
- 联调时能区分已接入与未接入
- 主流程不因美术缺失而卡死

---

## 4. 当前可后置的美术资产

以下内容可以后置，不进入当前优先制作范围：

- 大量次级角色差分
- 大量 NPC 个性化立绘
- 高复杂 UI 动效
- 场景大规模装饰铺满
- 多套战斗演出
- 非关键页面精修
- 额外国家 / 额外区域场景资产
- 大量 CG / 过场插图
- 深度养成相关图标体系
- 多敌人库储备
- 大量角色动作变体

---

## 5. 当前不建议开工的资产

以下内容当前不建议开工：

- 多国家场景包
- 大地图 / 大世界资产
- 大量支线专用角色资产
- 完整终端子页面全量资产
- 高规格战斗特效包
- 宣传向大图和完整物料
- 非主流程区域完整装修
- 长线成长系统相关视觉包

原因只有一条：

- 当前不服务主流程闭环

---

## 6. 美术开工包建议

## 6.1 P0：先开
1. 中央广场主舞台包
2. 蝶茧终端包
3. 对话与基础 UI 包
4. 关键角色包
5. 事件结果与反馈包

## 6.2 P1：主流程能跑后补
1. 次级 NPC 形象包
2. 场景补充装饰包
3. 终端细节强化包
4. 基础小特效包
5. 冲突 / 战斗表现强化包

## 6.3 P2：当前后置
1. 大量角色差分
2. 高复杂 UI 动效
3. 场景全面铺装
4. 多套冲突表现
5. 非关键页面精修
6. 额外区域资产
7. 宣传向 CG

## 7. 资产台账建议字段

后续建议按下表维护：

| 资产编号 | 资产名称 | 资产类别 | 使用位置 | 优先级 | 当前状态 | 跟进人 | 备注 |
|---|---|---|---|---|---|---|---|

字段说明：

- 资产编号：唯一编号
- 资产名称：资源名称
- 资产类别：场景 / 角色 / UI / 终端 / 事件反馈 / 战斗 / 特效 / 占位
- 使用位置：场景、页面或模块
- 优先级：P0 / P1 / P2
- 当前状态：未开始 / 制作中 / 可接入 / 已接入 / 需返修 / 已锁定
- 跟进人：当前主要跟进该资产的人

## 10. 当前结论

当前最需要的美术资产不是大体量内容，而是能把 Demo 主流程撑起来的核心资产：

- 主舞台
- 蝶茧终端
- 关键角色
- 基础 UI
- 事件反馈
- 1 次冲突表现

其余资产按 P1、P2 后置。
